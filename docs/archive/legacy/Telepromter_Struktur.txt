Struktur

Hier ist die vollständige, detaillierte Monorepo-Ordnerstruktur für das Projekt. Diese Struktur ist ideal für Tools wie **Turborepo** oder **Nx** in Kombination mit `pnpm workspaces`. Sie trennt die komplexen Domänen (KI-Pipeline, Datenbank, Rendering) strikt voneinander ab, was sowohl für menschliche Entwicklerteams als auch für KI-Coding-Agenten (wie GitHub Copilot) essenziell ist, um den Kontext nicht zu verlieren.

Diese Struktur bildet exakt das zuvor definierte Lastenheft (SRS) ab.

### Warum diese Struktur perfekt für dein Vorhaben ist:

1. **Sicherheit bei "Hot-Updates" (Live-Text-Editing):** Die Kernlogik für das Ruckelfreie Rendering (`pwa-prompter/src/webgl-renderer`) ist komplett isoliert von der UI der Redaktion (`pwa-director`). Wenn ein n8n-Webhook einen neuen Text in die Datenbank schiebt, verarbeitet das `backend-server` dies in Millisekunden und schickt nur ein winziges Payload via WebSocket an den Prompter, der es in der GPU rendert.
2. **Skalierbarkeit nach Tiers (Basic, Pro, Expert):** Braucht ein Basic-User kein `video-io` (NDI/ST2110), läuft das Backend ohne diese Module. Bei einem Tier-1 Broadcaster werden diese Module aktiviert.
3. **Plug-and-Play für die KI:** Das `pipeline-ai` Package sitzt zwischen Ingest und Datenbank. Man kann später problemlos von OpenAI auf eine komplett lokale Open-Source KI wechseln, ohne den Code im Backend oder Frontend anfassen zu müssen.
