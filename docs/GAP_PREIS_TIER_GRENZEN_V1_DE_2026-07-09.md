# GAP Preis- und Tiergrenzen v1

Stand: 2026-07-09
Ziel: Freigabefaehige v1-Entscheidungsvorlage fuer Basic, Professional und Expert.

## 1. Tier-Abgrenzung (v1)

- Basic:
  - Kern-Prompterbetrieb
  - Basis-Editor, Basis-Steuerung
  - Keine stoerende Werbung im Output-only Modus
- Professional:
  - Team-/Workflow-Erweiterungen
  - Erweiterte Titel-/Layout-/Profilfunktionen
  - Produktionsnahe Komfortfunktionen
- Expert:
  - Betriebskritische Integrationen
  - Fernsteuerungs-/Broadcast-nahe Features
  - Erweiterte Schnittstellen und Betriebskontrolle

## 2. Preislogik (v1-Entwurf)

- Monats- und Jahresmodell anbieten
- Jahresmodell mit rabattierter Effektivrate
- Pilotkunden zunaechst mit klar definierten Beta-Konditionen

## 3. Preisanker (Arbeitskorridor, noch nicht final)

- Basic:
  - Kostenlos oder niedriger Einstieg
  - Optionales Upsell-Modell ohne Output-Stoerung
- Professional:
  - Mittleres Preisband
  - Fokus auf Workflow-Nutzen
- Expert:
  - Hoechstes Preisband
  - Fokus auf Integrations- und Betriebssicherheit

## 4. Harte Grenzen (v1)

- Keine Preis-/Werbeentscheidung, die den Live-Output stoert
- Keine Tier-Ueberlappung bei Expert-Kernfeatures
- Keine finale Preisfreigabe ohne Interview-Validierung (mind. 5 Interviews)

## 5. Entscheidungsfragen fuer Freigabe

1. Bleibt Basic in der Beta kostenlos?
2. Welche 3 Funktionen trennen Professional klar von Basic?
3. Welche 3 Funktionen trennen Expert klar von Professional?
4. Welche Preisgrenzen gelten als Obergrenze pro Tier?
5. Wird ein zeitlich begrenzter Trial fuer Professional/Expert gesetzt?

## 6. Ergebnisformat fuer Freigabe-Meeting

- Tier-Matrix mit Feature-Grenzen (ja/nein)
- Preisband je Tier (min/max)
- Trial-Regel
- Go/No-Go fuer Beta-V1 Preisfreigabe
